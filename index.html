🔒 HTTPS SNI Server

Асинхронный сервер на Python с поддержкой SNI для обслуживания нескольких доменов с разными SSL-сертификатами на одном порту

🐍 Python 3.7+ 🔐 SNI Support ⚡ Asyncio 📱 Termux Ready 🆓 MIT License
🚀 Быстрый старт ⭐ GitHub

✨ Особенности

🔐
SNI Support
Разные сертификаты для разных доменов на одном порту
⚡
Asyncio
Асинхронная обработка для высокой производительности
📱
Termux
Работает на Android без root-прав (порт 8443)
🔧
Auto-load
Автоматическая загрузка сертификатов из папки
📄
HTML Response
Красивая адаптивная страница с именем домена
📝
Logging
Логирование всех подключений с указанием SNI

🚀 Быстрый старт

1. Клонировать репозиторий

git clone https://github.com/ваш-логин/https-sni-server.git
cd https-sni-server

2. Сгенерировать сертификаты

chmod +x setup.sh
./setup.sh

3. Запустить сервер

# Порт 8443 (не требует root)python3 server.py

# Или порт 443 (требует прав на Linux)
# sudo setcap 'cap_net_bind_service=+ep' $(which python3)
# python3 server.py  # с PORT = 443 в коде

4. Протестировать

# Тест site1.local
curl -k --resolve site1.local:8443:127.0.0.1 https://site1.local:8443/

# Тест site2.local
curl -k --resolve site2.local:8443:127.0.0.1 https://site2.local:8443/

# Проверка сертификата
curl -kv --resolve site1.local:8443:127.0.0.1 https://site1.local:8443/ 2>&1 | grep "subject:"
# Добавьте в /etc/hosts:
127.0.0.1 site1.local site2.local

# Затем откройте в браузере:
https://site1.local:8443/
https://site2.local:8443/

⚠️ Примите предупреждение о самоподписанном сертификате
# В Termux:
pkg install python openssl openssh

# Запустить сервер:
python3 server.py

# Доступ из локальной сети:
ifconfig  # узнать IP телефона
# С компьютера: https://192.168.1.XX:8443/

# Доступ из интернета:
ssh -R 80:localhost:8443 localhost.run

➕ Добавить свой домен

# Создать самоподписанный сертификат
openssl req -x509 -newkey rsa:2048 \
    -keyout certs/mydomain.local.key \
    -out certs/mydomain.local.crt \
    -days 365 -nodes \
    -subj "/CN=mydomain.local"

# Сервер подхватит автоматически!
# Тест:
curl -k --resolve mydomain.local:8443:127.0.0.1 https://mydomain.local:8443/
💡
Совет: Для продакшена используйте реальные сертификаты от Let's Encrypt: 
certbot certonly --standalone -d example.com
cp /etc/letsencrypt/live/example.com/fullchain.pem certs/example.com.crt
cp /etc/letsencrypt/live/example.com/privkey.pem certs/example.com.key

⚠️ Безопасность

🔐
Никогда не коммитьте приватные ключи (*.key) в публичный репозиторий!
Они уже исключены через .gitignore, но будьте внимательны.
Рекомендация Описание
*.key в .gitignore Приватные ключи не попадут в репозиторий
Let's Encrypt Используйте доверенные сертификаты для продакшена
Rate limiting Добавьте ограничение запросов при публичном доступе
Логирование Включите логирование для мониторинга атак
Порт >1024 Используйте порт 8443 для работы без root
